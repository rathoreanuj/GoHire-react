Application Workflows

Key user journeys across all three modules

Authentication Flow (All Modules)

1. Login / Signup

User enters email + password

2. Server validates credentials

Bcrypt compare • Generate OTP • Send to email

3. Two-Factor Authentication

User enters OTP from email

4. Token / Session Created

Admin: Session + Cookie • Applicant/Recruiter: JWT in localStorage

5. Redirect to Dashboard

Protected routes accessible

Job Posting & Application Flow

1. Recruiter registers company

Upload logo + proof document → Awaiting verification

2. Admin verifies company

Reviews proof document → Approves or rejects

3. Recruiter posts job/internship

Only verified companies can post • Auto-expires in 30 days

4. Applicant browses & applies

Search, filter, apply with resume • Data saved to applicant_db

5. Recruiter reviews applications

Priority sorted (pending + premium first) • Select or Reject

6. Applicant notified via email

EmailJS sends acceptance/rejection email

Premium Upgrade Flow (Applicant)

1. Navigate to Payment page

Choose plan: Monthly or Annual

2. Stripe Payment Intent created

Server creates intent → Client renders Stripe form

3. Process Payment

Stripe confirms → Server creates Premium_User + Receipt

4. View Receipt & Subscription

Premium features unlocked • Visible to admin via SOAP

Recruiter Pro Upgrade Flow

1. Navigate to Upgrade page

Recruiter Pro plan • ₹999/month

2. Stripe Checkout Session

Server creates checkout session → Redirect to Stripe

3. Verify & Activate

Verify session → Set isPremium = true on RecruiterUser

4. Premium Features

View full applicant profiles (cross-DB access)
